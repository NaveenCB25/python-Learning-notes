01 — Basics

What Is a Network?

A network refers to two or more connected computers that can share resources such as data, a printer, an Internet connection, applications, or a combination of these resources.

🏢

LAN — Local Area Network

Connects devices within a limited geographic area such as a building or campus.

🏙️

MAN — Metropolitan Area Network

Spans a city or metropolitan region, larger than a LAN but smaller than a WAN.

🌐

WAN — Wide Area Network

Covers broad geographic areas, interconnecting multiple LANs across cities or countries.

02 — Topologies

Network Topologies

—

Bus Topology

All devices share a single communication line (segment) terminated at both ends.

✦

Star Topology

All devices connect to a central hub. Simple to manage and fault-tolerant at node level.

❊

Extended Star Topology

Multiple star topologies linked together, forming a hierarchical network structure.

04 — OSI Model

The Seven Layers of the OSI Model

A layered network model that reduces complexity, standardizes interfaces, facilitates modular engineering, ensures interoperable technology, accelerates evolution, and simplifies learning.

7
Application
Network processes to applications; user-facing protocols (HTTP, FTP, SMTP)
6
Presentation
Data representation, encryption, compression (SSL/TLS, JPEG, ASCII)
5
Session
Inter-host communication; establishes, manages, and terminates sessions
4
Transport
End-to-end reliability; TCP (reliable) and UDP (unreliable); flow control; virtual circuits
3
Network
Logical IP addressing (32-bit); path selection; routing; QoS prioritization
2
Data Link
MAC addressing; packet creation/transmission; media access control; bridges & switches
1
Physical
Binary transmission; electrical & mechanical specifications; cables, connectors, signals

06 — Devices

Network Devices by OSI Layer

Hub / Repeater
Layer 1 — Physical. Repeats signals received on one port to all other ports. Acts as a central connection point (concentrator) but creates a shared collision domain.
Bridge / Switch
Layer 2 — Data Link. Examines the MAC address in data link headers and forwards frames only to the correct port, opening a full-bandwidth path between ports.
Router
Layer 3 — Network. Routes IP packets between networks, selecting the best path to deliver data to its destination across an internetwork.
Layer 3 Switch
Layer 3 — Network. Combines multiport Layer 2 switching with basic routing between VLANs. High-speed, low-latency alternative to traditional routers for intra-network traffic.
Multilayer Switch
Layers 2–4. Combines L2 switching, L3 routing, and L4 switching in a single device. Offers high-speed scalability and lower latency compared to traditional routers.
Gateway
Multi-layer. Hardware + software combination that connects dissimilar network environments by performing protocol translation across multiple OSI layers (e.g., PC network ↔ MAC network).
Firewall
Security device. A system or group of systems that manages and controls access between two or more networks (Inside, Outside, DMZ), enforcing security policies.
07 — Addressing

Network Addressing

🔢

MAC Address (Layer 2)

Hardware address burned into the NIC at the factory. Protocol-independent. Found on the MAC sublayer of the Data Link layer. Example: 00-0C-F1-5E-BE-F2

🌐

IP Address (Layer 3)

Logical, hierarchical 32-bit addressing scheme used by the Network layer. Defines how network layers of two hosts interact. Example: 192.168.6.17

Module Summary

  • Hub (L1) repeats signals to all ports; switch (L2) forwards only to the correct MAC port
  • Router (L3) selects best path for IP packets across an internetwork
  • Layer 3 switch combines L2 switching with basic inter-VLAN routing
  • OSI model has 7 layers: Physical → Data Link → Network → Transport → Session → Presentation → Application
  • LAN (building), MAN (city), WAN (global) define network geographic scope
  • Media types: twisted pair (cheap, 100 m), coaxial (500 m), fiber optic (60 km, fastest), wireless (54 Mbps, 100 m)
  • Firewall controls access between inside, outside, and DMZ networks
  • Gateway translates between dissimilar network environments across OSI layers
